# 比赛模块需求书 V1

## 1. 目标与范围

- 目标：搭建“赛前-赛中-赛后”完整闭环，沉淀可追溯历史数据，支持运营发奖与导出履约信息。
- 数据原则：全链路使用真实数据，不使用 mock。
- 适用对象：选手、队长、裁判、运营、系统管理员。

## 2. 业务流程总览

### 2.1 赛前
- 赛事创建：赛事名称、时间、赛制、地图池、积分规则、报名窗口。
- 报名功能：个人/战队报名、资料填写、重复报名拦截。
- 审核流程：待审核/通过/驳回，驳回原因必填。
- 名单冻结：开赛前锁定报名名单并生成冻结快照。

### 2.2 赛中
- 实时统计：每局击杀、伤害、排名、队伍积分、总积分榜。
- 进度管理：当前局次、局状态（未开始/进行中/已结束）。
- 异常处理：掉线、重赛、判罚、人工修正入口。
- 观赛展示：按局/按队伍/按选手维度切换数据。

### 2.3 赛后
- 自动结算：按规则计算最终名次与积分。
- 奖励发放：根据名次发放称号奖励（支持有效期、撤回）。
- 历史归档：赛事、对局、选手数据归档可检索。
- Excel 导出：导出成员姓名、电话、地址（含导出记录）。

## 3. 角色与权限

- 选手/队长：报名、查看赛程、查看战绩与奖励。
- 裁判：赛中异常处理、复核、判罚录入。
- 运营：赛事配置、名单审核、发奖、导出。
- 管理员：权限配置、参数配置、审计查看。

## 4. 查漏补缺（建议补充）

- 赛事状态机：草稿 -> 报名中 -> 进行中 -> 已结算 -> 已归档。
- 报名截止与名单冻结时间可配置，统一时区策略。
- 结算后设置“成绩复核窗口”（如 24 小时内可申诉）。
- 奖励发放幂等：重复触发不重复发称号。
- 导出权限分级：支持手机号/地址脱敏。
- 审计日志：报名审核、判罚、结算、发奖、导出全部可追踪。
- 兜底机制：实时数据中断时支持人工录入与回补。
- 性能指标：赛中榜单刷新延迟、并发规模、导出耗时上限。

## 5. 关键数据对象

- 比赛：matchId、赛制、规则版本、状态、时间窗口。
- 报名：报名主体、队伍成员、审核状态、冻结版本。
- 对局：局次、队伍表现、选手表现、异常事件。
- 结算：积分明细、名次、称号发放记录、回滚记录。
- 导出：导出人、导出字段、导出时间、下载记录。

## 6. MVP 验收标准

1. 报名-审核-冻结流程可完整跑通。
2. 赛中榜单可实时刷新且数据来源可追溯。
3. 赛后可自动结算并发放称号奖励。
4. 历史比赛支持按赛事/日期/选手检索。
5. 成员姓名/电话/地址可一键导出 Excel。
6. 全链路具备审计日志与异常记录。

## 7. 实施优先级建议

- P0（必须）：赛前报名审核、赛中实时统计、赛后结算发奖、历史归档、Excel 导出。
- P1（重要）：复核窗口、异常工单、导出脱敏、审计看板。
- P2（优化）：高级分析报表、自动异常检测、消息通知联动。
